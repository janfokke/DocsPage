Remarks

Files that are encapsulated inside a compiled assembly are commonly known as Manifiest or embedded resources. Since embedded resources are added to the assembly at compiled time, they can not be accidentally deleted or misplaced. However, if the file needs to be changed, the assembly will need to be re-compiled with the changed file.

To add an embedded resource file to an assembly, first add it to the project and then change the Build Action in the Properties of the file to 

Embedded Resource
. The next time the project is compiled, the compiler will add the file to the assembly as an embedded resource. The compiler adds namespace(s) to the embedded resource so it matches with the path of where the file was added to the project.
